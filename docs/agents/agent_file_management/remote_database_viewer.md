# Remote Database Viewer

## Purpose

Enables users to browse and query SQLite databases generated by agents directly in the browser, without external tools. SQLite files in the agent workspace are expandable in the environment panel and open a dedicated database viewer in a new tab.

## Core Concepts

- **Database Viewer** - Browser-based UI for browsing schema and querying SQLite databases
- **Auto Mode** - Read-only paginated browsing: auto-generates `SELECT * FROM table LIMIT/OFFSET`
- **Manual Mode** - Editable SQL textarea for executing any query (no pagination)
- **Schema Sidebar** - Expandable tree showing tables, views, and columns with types
- **Proxy Architecture** - Backend proxies all database requests to the agent-env, which reads SQLite files directly

## User Stories / Flows

### Browse Database Schema

1. User clicks a SQLite file (`.db`, `.sqlite`, `.sqlite3`) in the environment panel
2. File expands to show a list of tables (fetched from backend → agent-env)
3. User clicks on a specific table → database viewer opens in a new tab with that table pre-selected
4. Schema sidebar shows all tables/views with expandable column details (name, type, nullable, primary key)

### Auto Mode - Browse Table Data

1. Database viewer opens in Auto mode by default
2. SQL textarea shows auto-generated `SELECT * FROM {table}` (read-only)
3. Results table displays rows with sticky headers
4. Status bar shows row count, execution time, pagination controls
5. User navigates pages to browse large tables

### Manual Mode - Custom Queries

1. User toggles to Manual mode
2. SQL textarea becomes editable
3. User writes and executes any SQL query (SELECT, INSERT, UPDATE, DELETE)
4. Results display in the same table format
5. DML queries show `rows_affected` count instead of row data

### Export Results

1. User clicks CSV export button in the viewer
2. Current query results download as a CSV file

## Business Rules

- **Supported extensions:** `.db`, `.sqlite`, `.sqlite3`
- **Query modes:** Auto (paginated, read-only SQL) and Manual (any SQL, no pagination)
- **Default page size:** 1000 rows
- **Query timeout:** 30 seconds (configurable per request)
- **All SQL allowed** in Manual mode (not restricted to read-only)
- **Error handling:** Errors returned in response body (not HTTP errors) for frontend display; classified as `syntax_error`, `timeout`, `file_error`, `execution_error`
- **Connection management:** Each query opens and closes its own SQLite connection (no persistent connections)

## Architecture Overview

```
Environment Panel → Frontend Route (new tab) → Backend Proxy API → Agent-Env → SQLite File
                                                  (FastAPI)          (FastAPI)    (/app/workspace/)
```

The backend acts as a proxy - it forwards database requests to the agent environment, which reads SQLite files directly from the workspace.

## Integration Points

- **Agent File Management** - SQLite files are detected in the workspace file tree alongside regular viewable files. See [Agent File Management](./agent_file_management.md)
- **Agent Environments** - Database operations require a running environment; backend checks ownership. See [Agent Environments](../agent_environments/agent_environments.md)
- **Agent Environment Core** - Agent-env server implements the SQLite query engine, schema introspection, and path validation. See [Agent Environment Core](../agent_environment_core/agent_environment_core.md)
